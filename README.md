## WHY DELTADOCS?

If you run a RAG chatbot or AI agent over **someone else's docs**, you're stuck picking between two bad options:

```
  в”Ңв”Җ[ OPTION A ]в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ   в”Ңв”Җ[ OPTION B ]в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
  в”Ӯ  RE-CRAWL + RE-EMBED ALL      в”Ӯ   в”Ӯ  LET THE INDEX GO STALE       в”Ӯ
  в”Ӯ  ............................ в”Ӯ   в”Ӯ  ............................ в”Ӯ
  в”Ӯ  you pay for embeddings that  в”Ӯ   в”Ӯ  your bot confidently serves  в”Ӯ
  в”Ӯ  scale with TOTAL PAGES, even в”Ӯ   в”Ӯ  a deprecated API signature   в”Ӯ
  в”Ӯ  when almost nothing changed. в”Ӯ   в”Ӯ  or a long-removed flag.      в”Ӯ
  в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ   в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
                          вң— GAME OVER вң—
```

Great one-shot tools (Firecrawl, Crawl4AI) turn a URL into clean markdown. **None of them does cheap, scheduled _diffing_.** That's the gap DeltaDocs fills:

> рҹ”‘ Every chunk carries a **SHA-256 over _normalized_ text**, so a re-run emits **only the chunks that actually changed** вҖ” and ships them straight to your vector store.

Because the hash ignores whitespace and formatting noise, an unchanged page produces **zero** changes on re-run. You only ever re-embed real edits. вҳ… NEW HI-SCORE.

```
в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ
```

## QUICKSTART

> **Prerequisites:** Python **3.10+** and Git. _(Windows PowerShell shown; on macOS/Linux use `source .venv/bin/activate`.)_

```powershell
> git clone https://github.com/MayankSinghRaghav/DeltaDoc.git
> cd DeltaDoc
> python -m venv .venv
> .venv\Scripts\activate
> pip install -e ".[dev]"

> pytest -q                            # 40 passing вҖ” confirms your setup
```

```powershell
# 1UP  в”Җв”Җ one-shot crawl  ->  LLM-ready output
> deltadocs https://docs.python.org/3/ --max-pages 20 -o out
#    out\chunks.jsonl В· out\pages.jsonl В· out\llms.txt В· out\llms-full.txt

# 2UP  в”Җв”Җ change tracking  ->  run twice with a state dir; 2nd run writes the delta
> deltadocs https://docs.python.org/3/ --max-pages 20 --state-dir .state -o out
> deltadocs https://docs.python.org/3/ --max-pages 20 --state-dir .state -o out
#    out\changeset.json  ->  added / modified / removed chunks only
```

## HOW IT WORKS

```
        [ DOCS SITE ]
              в”Ӯ  crawl В· robots В· sitemap
              в–ј
        [ RAW HTML ]
              в”Ӯ  trafilatura
              в–ј
        [ CLEAN MARKDOWN ]
              в”Ӯ  heading split
              в–ј
        [ CHUNKS + SHA-256 ]
              в”Ӯ  diff vs saved state
              в–ј
        [ CHANGESET ]в•җв•җв•Ұв•җв•җв–ә  VECTOR DB  (Chroma / pgvector / Pinecone)
                       в• в•җв•җв–ә  WEBHOOK / SLACK
                       в• в•җв•җв–ә  LLM ENRICH  (summary + tags)
                       в•ҡв•җв•җв–ә  llms.txt
```

1. **Crawl** the site in-domain (sitemap + links), honoring robots.txt and your include/exclude globs.
2. **Extract** each page's main content to markdown and **chunk** it by heading.
3. **Hash** every chunk over _normalized_ text вҖ” the key insight that makes diffing trustworthy.
4. **Diff** this run's chunks against the previous run's saved state вҶ’ a `ChangeSet`.
5. **Deliver / enrich** вҖ” upsert changed chunks (delete removed) into your vector DB, summarize them, or ping Slack.

```
в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ в–Ҳв–Ҳв–Ҳв–Ҳ
```

## FEATURES

| | |
|---|---|
| рҹ•·пёҸ **Polite crawler** | `httpx` + sitemap/link discovery, robots.txt, include/exclude globs, depth вҖ” static/SSG docs, no headless browser |
| рҹ§ј **Clean extraction** | `trafilatura` strips nav/header/footer вҶ’ main-content markdown |
| вңӮпёҸ **Heading-aware chunks** | breadcrumb `heading_path`, stable `chunk_id`, token estimate |
| рҹ”‘ **Stable hashing** | SHA-256 over _normalized_ text вҶ’ trivial reformatting never looks like a change |
| рҹ”Җ **Delta engine** | diff vs the previous run вҶ’ `ChangeSet` of added / modified / removed |
| рҹҡҡ **Delivery** | push deltas to **Chroma / pgvector / Pinecone**, or alert via **webhook / Slack** |
| рҹ§  **Enrichment** | per-change LLM summary + tags (OpenAI-compatible **or** Anthropic/Claude) |
| рҹӨ– **MCP server** | expose `crawl_docs` / `diff_docs` as tools for AI agents |
| рҹ“„ **llms.txt** | generates `llms.txt` + `llms-full.txt` |
| вҳҒпёҸ **Two ways to run** | OSS Python CLI **and** a deployable Apify Actor (pay-per-event on changed chunks) |

## INTEGRATIONS

Optional, dependency-light. Wire deltas into your stack in a couple of lines:

```python
# Delivery вҖ” push the delta into your index, alert your team
from deltadocs.deliver import apply_changeset, open_chroma_collection, post_webhook

apply_changeset(changeset, open_chroma_collection("./chroma"))   # or PgVectorStore / PineconeVectorStore
post_webhook(changeset, "https://hooks.slack.com/вҖҰ")

# Enrichment вҖ” one-line summary + tags per change
from deltadocs.enrich import enrich_changeset, make_anthropic_summarizer

enriched = enrich_changeset(changeset, make_anthropic_summarizer())   # or make_openai_summarizer()
```
